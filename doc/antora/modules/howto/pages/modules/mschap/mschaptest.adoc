== Test the Server

Once the server is started in debugging mode, MS-CHAP authentication can be tested using `radtest`. The server must be configured with a known user and password before running any tests. In the examples below, we assume a user `bob` with password `hello`.

== Test MS-CHAPv1

[source,shell]
----
radtest -t mschap bob hello localhost 0 testing123
----

The `radtest` program computes the MS-CHAPv1 challenge and response, and places the correct `Vendor-Specific.Microsoft.CHAP-Challenge` and `Vendor-Specific.Microsoft.CHAP-Response` attributes in the packet.

The output of `radtest` should look like the following. Some numbers may differ, but the general format should be the same:

[source,log]
----
Sent Access-Request Id 23 from 0.0.0.0:40612 to 127.0.0.1:1812 length 79
	User-Name = "bob"
	NAS-IP-Address = 127.0.0.1
	NAS-Port = 0
	Message-Authenticator = 0x00
	Vendor-Specific.Microsoft.CHAP-Challenge = 0x696bcaff8f8bef29
	Vendor-Specific.Microsoft.CHAP-Response = 0x000100000000000000000000000000000000000000000000000016c32819add27b3d29f6866506e6cc6548f50b6429518579
Received Access-Accept Id 23 from 127.0.0.1:1812 to 127.0.0.1:40612 length 44
	Vendor-Specific.Microsoft.MPPE-Encryption-Policy = Encryption-Allowed
	Vendor-Specific.Microsoft.MPPE-Encryption-Types = RC4-40or128-bit-Allowed
----

A successful MS-CHAPv1 authentication returns an `Access-Accept` containing `MPPE-Encryption-Policy` and `MPPE-Encryption-Types` attributes. These MPPE key attributes are generated by the `mschap` module after computing the session key.

== Test MS-CHAPv2

[source,shell]
----
radtest -t mschapv2 bob hello localhost 0 testing123
----

MS-CHAPv2 uses a 16-byte challenge and provides mutual authentication. The server includes a `CHAP2-Success` attribute in the `Access-Accept` that the client verifies.

The output of `radtest` should look like the following:

[source,log]
----
Sent Access-Request Id 47 from 0.0.0.0:52132 to 127.0.0.1:1812 length 109
	User-Name = "bob"
	NAS-IP-Address = 127.0.0.1
	NAS-Port = 0
	Message-Authenticator = 0x00
	Vendor-Specific.Microsoft.CHAP-Challenge = 0x16d2833f4239256dd2b2bb26f2ecb2a3
	Vendor-Specific.Microsoft.CHAP2-Response = 0x0001502feeee9495a353cddbd1efc40072820000000000000000e866286bb30d0215ed16cf425b6a29d206667a9853e23ca4
Received Access-Accept Id 47 from 127.0.0.1:1812 to 127.0.0.1:52132 length 84
	Vendor-Specific.Microsoft.CHAP2-Success = 0x00533d...
	Vendor-Specific.Microsoft.MPPE-Encryption-Policy = Encryption-Allowed
	Vendor-Specific.Microsoft.MPPE-Encryption-Types = RC4-40or128-bit-Allowed
----

The `CHAP2-Success` attribute in the reply is the server authenticator response (beginning with `S=`). A non-empty `CHAP2-Success` confirms that mutual authentication succeeded.

== Read the server debug output

Read the server debug output and look for messages containing `ERROR` or `WARNING`, or the `mschap` module name.

Representative debug output for a successful MS-CHAPv2 authentication:

[source,log]
----
(0)  Received Access-Request ID 47 from 127.0.0.1:52132 to 127.0.0.1:1812 length 109 via socket proto_radius_udp server * port 1812
(0)    User-Name = "bob"
(0)    Vendor-Specific.Microsoft.CHAP-Challenge = 0x16d2833f4239256dd2b2bb26f2ecb2a3
(0)    Vendor-Specific.Microsoft.CHAP2-Response = 0x0001...
...
(0)    mschap - Found MS-CHAPv2
(0)    mschap - Setting control.Auth-Type = mschap
(0)    mschap (ok)
...
(0)  Found "known good" password in control.Password.Cleartext
(0)  Running 'authenticate mschap' from file ./raddb/sites-enabled/default
(0)  authenticate mschap {
(0)    mschap - Authenticating with MS-CHAPv2
(0)    mschap - MS-CHAPv2 authentication succeeded
(0)    mschap (ok)
(0)  } # authenticate mschap (ok)
...
(0)  Sending Access-Accept ID 47 from 127.0.0.1:1812 to 127.0.0.1:52132 length 84 via socket proto_radius_udp server * port 1812
(0)    Vendor-Specific.Microsoft.CHAP2-Success = 0x00533d...
(0)    Vendor-Specific.Microsoft.MPPE-Encryption-Policy = Encryption-Allowed
(0)    Vendor-Specific.Microsoft.MPPE-Encryption-Types = RC4-40or128-bit-Allowed
...
----

The `mschap` module runs in two phases. In `recv Access-Request { ... }` it detects the MS-CHAP attributes and sets `control.Auth-Type := mschap`. In `authenticate mschap { ... }` it retrieves the known-good password set by a previous module (`files`, `sql`, etc.) and performs the MS-CHAP cryptographic calculations.

[NOTE]
====
The server unit tests in `src/tests/modules/mschap/` contain reference challenge and response byte vectors for both MS-CHAPv1 (`auth.attrs`, `auth.unlang`) and MS-CHAPv2 (`auth_mschapv2.attrs`, `auth_mschapv2.unlang`). These can be used to verify that the module is computing correct cryptographic results independently of `radtest`.
====
