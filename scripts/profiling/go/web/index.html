▸i Info

CEst (CPU Cycle Estimate) weights each callgrind counter by its approximate cycle cost:

CEst = Ir + 10*(I1mr+D1mr+D1mw) + 100*(ILmr+DLmr+DLmw) + 10*(Bcm+Bim)

Ir Instruction Fetch  ·  I1mr L1 Instr. Fetch Miss  ·  D1mr L1 Data Read Miss  ·  D1mw L1 Data Write Miss  ·  ILmr LL Instr. Fetch Miss  ·  DLmr LL Data Read Miss  ·  DLmw LL Data Write Miss  ·  Bcm Mispredicted Cond. Branch  ·  Bim Mispredicted Ind. Branch

Two ways to read the heatmap

  1. One run — cost view. Functions are ranked by self-CEst, most expensive on top, and coloured on a heat ramp (worst → cheapest) so you see where the cycles go. Pick the worst-end colour in the legend.
  2. Two or more runs — change view. Choose one run as the baseline (the first loaded run by default; click set base on any chip to change it). Every cell's Δ is measured against that run, per function:
Δ%(fn, run) = ( CEstfn,run − CEstfn,base ) / CEstfn,base × 100

There is no mean/median baseline: for change-over-time you compare against a real reference run you pick, not a synthetic average.
